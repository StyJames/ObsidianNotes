1.为什么不一次全部加载到容器中

如果Spring没有添加任何的附加条件，此时这些配置类中所定义的bean都会被导入到Spring容器中，这样
非常消耗内存，因此在Spring中提供了很多的条件注解，通过这些条件注解控制某一个配置是否生效。

2.简单总结:

在项目启动的时候，Spring Boot框架会自动读取META-INF/spring.factories配置文件中org.springframework.boot.autoconfigure.EnableAutoConfiguration所配置的配置类，然后将其中所定义的bean根据条件注解所指定的条件来决定是否需要将其导入到Spring容器中。

3.详细总结

  1. 在Spring Boot项目中有一个注解@SpringBootApplication，这个注解是对三个注解进行了封装：@SpringBootConfiguration、@EnableAutoConfiguration、@ComponentScan
    其中@EnableAutoConfiguration是实现自动化配置的核心注解。
  2. 该注解通过@Import注解导入AutoConfigurationImportSelector，这个类实现了一个导入器接口ImportSelector。在该接口中存在一个方法selectImports，
  3. 该方法的返回值是一个数组，数组中存储的就是要被导入到spring容器中的类的全类名。在AutoConfigurationImportSelector类中重写了这个方法,
  4. 该方法内部就是读取了项目的classpath路径下META-INF/spring.factories文件中的所配置的类的全类名。
  5. 在这些配置类中所定义的Bean会根据条件注解所指定的条件来决定是否需要将其导入到Spring容器中。

Spring Boot自动装配是基于Spring框架的基础上实现的，其本质是通过注解和配置文件来实现的。Spring Boot通过自动扫描和装配的方式，自动加载并配置应用程序中所需的组件，减少了开发人员的工作量，提高了开发效率。 Spring Boot自动装配的原理主要包括以下几个方面： 1. 自动扫描：Spring Boot会自动扫描应用程序中的所有组件，包括Java类、XML配置文件、注解等。 2. 自动配置：Spring Boot会根据应用程序中的组件信息自动配置应用程序的运行环境，包括数据源、Web应用程序等。 3. 条件装配：Spring Boot会根据应用程序的运行环境自动选择合适的组件进行装配，例如根据运行环境自动选择不同的数据库连接池。 4. 预定义配置：Spring Boot会提供一些预定义的配置，例如默认的数据源配置、默认的Web应用程序配置等，可以通过修改配置文件或者自定义注解来进行修改。 总之，Spring Boot的自动装配原理是通过自动扫描、自动配置、条件装配和预定义配置等方式来实现的，使得开发人员可以更加方便快捷地开发应用程序。
